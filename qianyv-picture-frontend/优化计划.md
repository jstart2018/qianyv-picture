## 二、优化目标与原则

### 2.1 核心目标

1. 文件轻量化： 单个文件不超过 500 行（建议 300-400 行）

1. 代码复用性： 相同功能封装为公共组件/工具函数

1. 可维护性提升： 模块职责清晰、命名规范统一

1. 功能完整性： 保证原有功能和样式 100% 一致

### 2.2 拆分标准

#### JS 拆分标准（按职责）：

```text
组件拆分：
├── UI 组件：纯展示组件（如卡片、按钮、标签）
├── 业务组件：包含业务逻辑（如上传模块、列表模块）
├── Hooks：可复用的逻辑（如 usePagination、useUpload）
└── Utils：纯函数工具（如格式化、验证）

Composables（组合式函数）：
├── useSpace.ts - 空间相关逻辑
├── useUpload.ts - 上传相关逻辑
├── usePagination.ts - 分页逻辑
└── useDebounce.ts - 防抖/节流
```

#### CSS 拆分标准（按用途）：

```text
样式拆分：
├── base.css - 全局基础样式（重置、变量）
├── theme.css - 主题配置（颜色、间距、圆角）
├── animations.css - 动画效果
├── components/ - 组件样式
│   ├── button.css - 按钮样式
│   ├── card.css - 卡片样式
│   ├── form.css - 表单样式
│   └── loading.css - 加载状态
└── pages/ - 页面专属样式（按需保留）
```

### 2.3 命名规范

#### 文件命名：

- 组件： PascalCase（如 UserCard.vue、ImageUploader.vue）

- Hooks： camelCase + use 前缀（如 useAuth.ts、usePagination.ts）

- 工具函数： camelCase（如 formatDate.ts、parseTag.ts）

- 样式文件： kebab-case（如 button-styles.css、card-styles.css）

#### 代码命名：

- 变量/函数： camelCase（如 currentPage、fetchData）

- 常量： UPPER_SNAKE_CASE（如 MAX_FILE_SIZE、API_BASE_URL）

- CSS 类名： kebab-case（如 .user-card、.loading-spinner）

```ts
/**
 * 格式化文件大小
 * @param bytes 文件字节数
 * @returns 格式化后的字符串（如 "2.5 MB"）
 */
export function formatFileSize(bytes: number): string {
  if (!bytes || bytes === 0) return '0 B'

  const units = ['B', 'KB', 'MB', 'GB', 'TB']
  const k = 1024
  const i = Math.floor(Math.log(bytes) / Math.log(k))
  const size = bytes / Math.pow(k, i)

  return `${size.toFixed(2)} ${units[i]}`
}
```

```ts
/**
 * 格式化日期
 * @param dateString 日期字符串
 * @param format 格式（默认 'YYYY-MM-DD'）
 * @returns 格式化后的日期字符串
 */
export function formatDate(
  dateString: string | Date,
  format: 'full' | 'date' | 'time' = 'date',
): string {
  if (!dateString) return '未知'

  const date = new Date(dateString)
  if (isNaN(date.getTime())) return '无效日期'

  const options: Intl.DateTimeFormatOptions = {
    date: { year: 'numeric', month: '2-digit', day: '2-digit' },
    time: { hour: '2-digit', minute: '2-digit', second: '2-digit' },
    full: {
      year: 'numeric',
      month: '2-digit',
      day: '2-digit',
      hour: '2-digit',
      minute: '2-digit',
      second: '2-digit',
    },
  }[format]

  return date.toLocaleString('zh-CN', options)
}
```

```ts
/**
 * 解析标签数据
 * @param tagsData 标签数据（可能是字符串、数组或 JSON）
 * @returns 标签数组
 */
export function parseTags(tagsData: string | string[] | null | undefined): string[] {
  if (!tagsData) return []

  try {
    // 如果已经是数组，直接返回
    if (Array.isArray(tagsData)) {
      return tagsData.filter((tag) => typeof tag === 'string' && tag.trim())
    }

    // 如果是 JSON 数组字符串，解析它
    if (typeof tagsData === 'string' && tagsData.startsWith('[')) {
      const parsed = JSON.parse(tagsData)
      if (Array.isArray(parsed)) {
        return parsed.filter((tag) => typeof tag === 'string' && tag.trim())
      }
    }

    // 如果是逗号分隔的字符串
    if (typeof tagsData === 'string') {
      return tagsData
        .split(',')
        .map((tag) => tag.trim())
        .filter((tag) => tag)
    }

    return []
  } catch (err) {
    console.error('解析标签失败:', err)
    return []
  }
}
```

```ts
import { FILE_CONSTRAINTS } from '@/constants/file'

export interface FileValidationResult {
  valid: boolean
  error?: string
}

/**
 * 验证文件大小
 */
export function validateFileSize(file: File): FileValidationResult {
  const maxSize = file.type.startsWith('image/')
    ? FILE_CONSTRAINTS.MAX_IMAGE_SIZE
    : FILE_CONSTRAINTS.MAX_VIDEO_SIZE

  if (file.size > maxSize) {
    const maxSizeMB = maxSize / (1024 * 1024)
    return {
      valid: false,
      error: `文件过大，最大支持 ${maxSizeMB}MB`,
    }
  }

  return { valid: true }
}

/**
 * 验证文件类型
 */
export function validateFileType(file: File, allowedTypes: string[]): FileValidationResult {
  if (!allowedTypes.includes(file.type)) {
    return {
      valid: false,
      error: `不支持的文件类型：${file.type}`,
    }
  }

  return { valid: true }
}
```

```ts
import { ref, computed } from 'vue'

export interface UsePaginationOptions {
  pageSize?: number
  initialPage?: number
}

export function usePagination(options: UsePaginationOptions = {}) {
  const current = ref(options.initialPage || 1)
  const pageSize = ref(options.pageSize || 20)
  const total = ref(0)
  const loading = ref(false)

  const hasMore = computed(() => {
    return current.value * pageSize.value < total.value
  })

  const totalPages = computed(() => {
    return Math.ceil(total.value / pageSize.value)
  })

  const nextPage = () => {
    if (hasMore.value) {
      current.value++
    }
  }

  const prevPage = () => {
    if (current.value > 1) {
      current.value--
    }
  }

  const reset = () => {
    current.value = 1
    total.value = 0
  }

  return {
    current,
    pageSize,
    total,
    loading,
    hasMore,
    totalPages,
    nextPage,
    prevPage,
    reset,
  }
}
```

文件：src/composables/useDebounce.ts

```ts
import { ref } from 'vue'

export function useDebounce<T extends (...args: any[]) => any>(fn: T, delay: number = 300) {
  let timer: ReturnType<typeof setTimeout> | null = null

  const debouncedFn = (...args: Parameters<T>) => {
    if (timer) clearTimeout(timer)

    timer = setTimeout(() => {
      fn(...args)
    }, delay)
  }

  const cancel = () => {
    if (timer) {
      clearTimeout(timer)
      timer = null
    }
  }

  return {
    debouncedFn,
    cancel,
  }
}
```

## 七、可行性分析与改进建议

### 7.1 当前方案存在的问题

#### ⚠️ 问题1：样式全局化方案可能导致冲突

**原方案：** 提取公共样式到全局 CSS 文件
**问题：** 项目已使用 scoped style，全局样式会破坏样式隔离
**改进方案：**

- 保留 scoped style 作为主要方案
- 仅提取 CSS 变量（:root）到全局
- 重复样式通过 **组件复用** 而非全局类解决

#### ⚠️ 问题2：执行顺序不够优化

**原方案：** 先样式 → 工具函数 → Hooks → 大文件拆分
**问题：** 应该先解决最紧急的重复代码
**改进方案：**

```txt
优化后的执行顺序：
1. 提取工具函数（1天） - 最简单、无风险
2. 合并横竖屏详情页（2天） - 90%重复代码，收益最大
3. 提取 Composables（2天） - 为大文件拆分做准备
4. 拆分 SpaceView（3天） - 最复杂的部分
5. 拆分其他大文件（3天）
6. 提取公共组件（2天）
```

#### ⚠️ 问题3：缺少路由配置调整说明

**原方案：** 未提及路由变化
**问题：** 横竖屏合并后，路由需要调整
**改进方案：**

```typescript
// router/index.ts - 修改路由配置
{
```
