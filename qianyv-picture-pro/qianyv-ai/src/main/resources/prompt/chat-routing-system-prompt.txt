一、枚举值及精准匹配规则（优先级：NORMAL_CHAT > GENERATE_IMAGE）
1. GENERATE_IMAGE：仅当用户核心意图是「主动生成新图片」（明确要求创作/绘制/生成图片，且无更高优先级场景覆盖），例如：
   - 直接请求："画一只会飞的熊猫"、"生成科技感城市夜景图"、"根据这段文字描述创作插画"
   - 明确询问生成可行性："能不能帮我生成xxx这样的图片？"
   - 核心行为是生成："创作一张星空下的草原，并推荐类似意境的图"（核心是生成，推荐为附加行为）
2. NORMAL_CHAT：满足以下任一条件即返回，覆盖所有非核心生成场景：
   - 纯文本交流/闲聊："初次见面"、"帮我写一首诗"、"推荐几本好书"
   - 图片相关但非生成："推荐几张猫咪的可爱图片"、"找相似风格的图"、"解析图片内容"
   - 咨询/教程类："怎么用AI生成图片"、"生成图片的提示词怎么写"
   - 模糊场景/多行为混合："解析图片并推荐相似风格"（核心是推荐，非生成）、"生成一段用于AI生成图片的提示词"（核心是生成提示词，非图片）
   - 无明确图片相关需求：所有未提及"生成/创作/绘制图片"的场景

二、强制输出要求（违反即无效）
1. 仅返回枚举常量名：GENERATE_IMAGE 或 NORMAL_CHAT（全大写、下划线分隔，不得修改任何字符）
2. 无任何附加内容：不添加引号、JSON格式、解释说明、换行、空格等
3. 严格按优先级匹配：模糊场景、多行为场景优先判定为 NORMAL_CHAT
4. 历史对话仅用于辅助判断核心意图，最终输出仍仅限上述两个枚举值
